# MyBlog 前端架构说明

本文档说明本项目的**前端架构、技术选型、页面组成、布局复用设计**，重点剖析页脚（Footer）等公共区域的 HTML / CSS / JS 组合方式。

---

## 1. 技术栈总览

| 维度 | 选型 | 说明 |
| --- | --- | --- |
| 视图层 | **Thymeleaf**（服务端渲染 / SSR） | Spring Boot 原生模板引擎，`application.yml` 配置了 `prefix: classpath:/templates/`、`suffix: .html` |
| 样式 | 原生 **CSS3**（单文件 `blog.css`） | 使用 CSS 变量、Grid、Flex、动画、毛玻璃、媒体查询 |
| 脚本 | 原生 **JavaScript**（ES6+） | 无框架，基于 `DOMContentLoaded` + `IntersectionObserver` + `fetch` |
| 图标 | **RemixIcon**（CDN 字体图标） | `remixicon@4.3.0`，以 `<i class="ri-...">` 形式使用 |
| 富文本 | **EasyMDE**（CDN Markdown 编辑器） | 仅后台文章编辑弹窗使用 |
| 后端 | Spring Boot + MyBatis | 提供页面路由（Controller 返回模板名）与 REST API（`/api/**`） |

> 整体定位：**服务端渲染的多页应用（MPA）**，没有前端构建工具（无 webpack/vite/Node 构建步骤），所有资源直接由 Spring Boot 静态托管。

---

## 2. 整体架构思路

```
浏览器请求 "/"
      │
      ▼
Spring Boot Controller (HomeController 等)
      │  ① 查询数据，放入 Model
      ▼
Thymeleaf 模板渲染
      │  ② 通过 th:replace 拼接 fragments 片段
      │  ③ 注入 Model 数据（文章列表、统计等）
      ▼
完整 HTML 响应  ──►  浏览器执行 blog.js / admin.js
                          │
                          ▼
                    fetch 调用 /api/**（点赞、评论、CRUD）
```

核心设计原则：**公共结构只写一次**。导航栏、页脚、背景、`<head>`、Hero 区域都被抽离到 `fragments.html` 中，通过 Thymeleaf 的 **片段（Fragment）机制** 在其它页面复用。

## 4. 布局复用机制（核心）

所有公共区域定义在 `src/main/resources/templates/fragments.html`，通过 `th:fragment` 声明、通过 `th:replace="~{fragments :: 片段名}"` 引用。

### 4.1 可用的片段清单

| 片段名 | 用途 | 是否带参数 |
| --- | --- | --- |
| `siteHead` | `<head>`：字符集、视口、CSRF meta、引入 `blog.css` 与 RemixIcon | 否 |
| `siteBackground` | 星空粒子容器 `.stars-container` | 否 |
| `siteGridBg` | 网格背景容器 `.grid-bg` | 否 |
| `siteNavbar(active)` | 顶部固定导航栏，参数 `active` 控制高亮菜单 | 是 |
| `heroSection(title, subtitle, showStats, showContactBtn)` | Hero 头图区，4 个参数控制标题/副标题/统计/联系按钮 | 是 |
| `siteFooter` | ★ 页脚 | 否 |

### 4.2 片段定义示例（fragments.html）

```html
<footer th:fragment="siteFooter" class="site-footer" id="footer">
    <div class="footer-inner">
        <div class="footer-col"> ... </div>
        <div class="footer-col"> ... </div>
        <div class="footer-col"> ... </div>
    </div>
    <div class="footer-bottom">© ...</div>
</footer>
```

### 4.3 页面引用示例（index.html）

```html
<head>
    <title th:text="${title} + ' - MyBlog'">首页 - MyBlog</title>
    <th:block th:replace="~{fragments :: siteHead}"></th:block>
</head>
<body>
    <th:block th:replace="~{fragments :: siteBackground}"></th:block>
    <th:block th:replace="~{fragments :: siteGridBg}"></th:block>
    <th:block th:replace="~{fragments :: siteNavbar('home')}"></th:block>
    <th:block th:replace="~{fragments :: heroSection(${title}, '分享技术心得…', true, false)}"></th:block>

    <!-- 各页面独立主体 -->
    ...
    <th:block th:replace="~{fragments :: siteFooter}"></th:block>

    <script th:src="@{/js/blog.js}"></script>
</body>
```

通过这种「**片段拼装**」模式，新增页面只需声明 `<head>`、引用片段、编写自己的主体即可，导航栏与页脚的修改一处生效、全站同步。

## 5. 页面清单与设计

### 5.1 首页 `index.html`
- 路由：`GET /`（HomeController.home）
- 结构：`Hero` → `主内容区(.main-container > .content-grid)`（左：文章卡片列表 `article-card`；右：侧边栏 `aside` 含个人卡片 / 分类标签云 / 热门文章）→ `页脚`
- 数据绑定：`articles`（文章列表）、`hotArticles`（热门文章）、`articleCount / totalViews / totalLikes`（Hero 统计）
- 交互：文章卡片由 Thymeleaf `th:each` 服务端循环渲染

### 5.2 关于我 `about.html`
- 路由：`GET /about`
- 结构：`Hero(showStats=false, showContactBtn=true)` → `关于主体(.about-wrapper)`（个人简介 / 技能条 `.skill-bar` / 时间线 `.timeline`）→ `页脚`
- 特色：技能条宽度由内联 `style="width: 90%"` 控制，加载时由 `blog.js` 用 `IntersectionObserver` 触发动画

### 5.3 文章详情 `article/detail.html`
- 路由：`GET /article/{id}`（草稿或不存在返回 `error/404`）
- 结构：导航栏（**内联写死，未复用片段**）→ 文章头（分类/标题/元信息）→ `正文(.article-detail-body，th:utext 输出 HTML)` → 点赞按钮 → 评论区 → `页脚（内联写死）`
- 交互（blog.js）：点赞 `POST /api/like/{id}`、评论加载与提交 `/api/comments/{id}`，含 XSS 转义 `escapeHtml()`

### 5.4 登录 `login.html`
- 路由：`GET /login`
- 结构：导航栏（复用 `siteNavbar('login')`）→ 登录卡片（用户名/密码表单，`POST /login` 由 Spring Security 处理）→ 页脚

### 5.5 后台 `admin/dashboard.html`
- 路由：`GET /admin`（需 ADMIN 权限）
- 结构：导航栏 → 后台 Hero + 统计卡片 → 文章管理表格 → 三个弹窗（新建/编辑文章、删除确认、修改密码）→ 页脚
- 额外资源：引入 EasyMDE 的 CSS/JS，以及 `admin.js`
- 交互（admin.js）：基于 `fetch` 的增删改查，编辑使用 EasyMDE Markdown 编辑器（`POST /admin/api/articles` 等）

### 5.6 错误页 `error/{404,500,default}.html`
- 结构：星空+网格背景 + 居中错误码大字 + 返回首页按钮
- 注意：**未复用 `fragments`，页脚也未出现**，样式以内联 `<style>` 为主，仅引用 `blog.css` 的变量与背景类

## 6. 页脚（Footer）复用详解 ★

这是用户重点关注部分，下面从 **HTML 结构、CSS 样式、JS 交互、组合方式** 四个角度拆解。

### 6.1 HTML 结构（`fragments.html` 中定义一次）

```html
<footer th:fragment="siteFooter" class="site-footer" id="footer">
    <div class="footer-inner">                       <!-- 三列网格 -->
        <div class="footer-col">
            <h4>关于本站</h4>
            <p>一个基于 Spring Boot + MyBatis + Thymeleaf 构建的个人博客…</p>
        </div>
        <div class="footer-col">
            <h4>快速链接</h4>
            <a href="/">首页</a>
            <a href="/about">关于我</a>
        </div>
        <div class="footer-col">
            <h4>联系方式</h4>
            <a href="mailto:...@gmail.com"><i class="ri-mail-line"></i> ...@gmail.com</a>
        </div>
    </div>
    <div class="footer-bottom">                     <!-- 版权条 -->
        &copy; 2026 MyBlog. Powered by Spring Boot. All rights reserved.
    </div>
</footer>
```

要点：
- 语义化 `<footer>` 标签 + `id="footer"`，供导航栏「联系」链接与 Hero「联系我」按钮通过 `#footer` 锚点平滑滚动定位
- 三列 `.footer-col`：关于 / 链接 / 联系，联系列使用 RemixIcon 图标 `<i class="ri-mail-line">`
- 内容中的文案、邮箱为**硬编码**（非动态数据）

### 6.2 CSS 样式（`blog.css` 中 `页脚` 段，约 1021–1069 行）

```css
.site-footer {
    background: var(--bg-secondary);
    border-top: 1px solid var(--border-color);
    padding: 48px 24px 24px;
    position: relative;
    z-index: 1;                 /* 浮于背景之上 */
}
.footer-inner {
    max-width: 1200px;          /* 与导航栏/主容器同宽对齐 */
    margin: 0 auto;
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;   /* 三列等宽 */
    gap: 40px;
}
.footer-col h4 { font-size: .95rem; font-weight: 700; ... }
.footer-col p, .footer-col a { font-size: .85rem; color: var(--text-secondary); ... }
.footer-col a:hover { color: var(--accent-light); }
.footer-bottom {
    max-width: 1200px; margin: 32px auto 0;
    padding-top: 20px; border-top: 1px solid var(--border-color);
    text-align: center; font-size: .8rem; color: var(--text-muted);
}
```

设计特征：
- 全部颜色取自 **CSS 变量**（暗色主题），与全站统一
- 三列 `grid` 布局，移动端（`@media max-width:900px`）降为单列
- 顶部 `border-top` 分隔线 + `z-index:1` 确保位于星空/网格背景（z-index:0）之上

### 6.3 JS 交互

页脚本身 **没有专属 JS 逻辑**。其「联系」锚点滚动由通用 `blog.js` 统一处理：

```javascript
// blog.js：所有 a[href^="#"] 平滑滚动
document.querySelectorAll('a[href^="#"]').forEach(anchor => {
    anchor.addEventListener('click', function (e) {
        const target = document.querySelector(this.getAttribute('href'));
        if (target) { e.preventDefault(); target.scrollIntoView({ behavior:'smooth' }); }
    });
});
```

即点击导航栏/ Hero 的「联系」→ 平滑滚动到 `<footer id="footer">`。

### 6.4 组合方式（如何被复用）

调用侧统一使用一行 `th:replace`：

```
index.html        → <th:block th:replace="~{fragments :: siteFooter}"></th:block>
about.html        → 同上
login.html        → 同上
admin/dashboard.html → 同上
```

渲染时 Thymeleaf 把 `fragments.html` 中的整段 `<footer>` 原样替换进页面，**无需任何额外 CSS/JS 引入**（样式已在 `blog.css`、脚本已在 `blog.js` 中）。

### 6.5 ✅ 页脚复用（已统一修复）

- 早期 `article/detail.html` 与 `error/*.html` 内联复制了页脚副本且邮箱/年份不一致（`admin@example.com`、`© 2024`）。**现已统一改为 `th:replace="~{fragments :: siteFooter}"`**，全站页脚唯一来源。
- 站点名、联系邮箱、版权年份均已**抽取为可配置变量**：由 `GlobalModelAdvice`（`@ControllerAdvice`）向所有页面模型注入 `siteName` / `siteEmail` / `copyrightYear`，值来自 `application.yml` 的 `blog.site.*`，年份自动取当前年。模板片段中保留兜底默认值，避免属性缺失导致空白。
- 详情页导航栏也已改为复用 `siteNavbar` 片段，与首页/关于/登录/后台保持一致。

## 7. 主体区域（Body 主体）设计

各页面主体均遵循「**背景层(z0) → 内容层(z1)**」的分层：

| 层级 | 元素 | z-index | 说明 |
| --- | --- | --- | --- |
| 背景层 | `.stars-container`（JS 注入 80 颗星）、`.grid-bg`（CSS 网格+径向遮罩） | 0 | `position:fixed`，全屏铺底 |
| 内容层 | 导航栏 `navbar`(fixed, z1000) / Hero / 主容器 / 页脚 | 1 | `position:relative; z-index:1` |

主体布局模式：
- **首页/关于**：`max-width:1200px` 居中容器 + 内部分栏（`content-grid` 为 `1fr 340px` 双列，关于页为单栏 `820px`）
- **文章详情**：`max-width:820px` 沉浸阅读栏，正文使用富文本排版样式（h1-h6 / 引用 / 代码块 / 表格）
- **后台**：`1200px` 容器 + 统计卡片 `grid(3列)` + 表格 + 弹窗（`position:fixed` 遮罩层）

## 8. CSS 设计系统

`blog.css` 是一个自给自足的设计系统，关键构成：

1. **设计令牌（CSS 变量）**：`:root` 中定义颜色（暗色系）、渐变 `--gradient-1/2/3`、圆角 `--radius-*`、阴影 `--shadow-*`、过渡 `--transition`。全站颜色/间距统一引用变量。
2. **基础重置**：`* { margin:0; padding:0; box-sizing:border-box }`，`body` 设置字体栈、暗色背景、抗锯齿。
3. **毛玻璃 & 渐变**：卡片用 `rgba(...,0.7)` + `backdrop-filter: blur()`；标题/按钮用 `linear-gradient` + `background-clip:text` 实现渐变文字。
4. **动画**：`@keyframes` 定义星星闪烁、Hero 浮动、头像脉冲、评论淡入、弹窗进入；工具类 `.fade-up` 配合 `IntersectionObserver` 实现滚动渐显。
5. **响应式**：两档断点 `@media (max-width:900px)`（双列→单列、导航变汉堡菜单）与 `@media (max-width:640px)`（缩小间距与字号）。
6. **暗色滚动条**：`::-webkit-scrollbar` 自定义。

## 9. JavaScript 模块设计

### 9.1 `blog.js`（全站通用）
包裹在 `DOMContentLoaded` 内，按「功能块」组织，使用**特性检测**（先 `querySelector` 存在才绑定）：
- 导航栏滚动加 `.scrolled` 类（`window.scroll`）
- 移动端汉堡菜单开合
- 滚动渐显 `IntersectionObserver`（`.fade-up`）
- 技能条宽度动画 `IntersectionObserver`（`.skill-bar-fill`）
- 星空背景注入（循环创建 80 个 `.star` 并随机位置/大小/动画时长）
- 点赞按钮（`fetch /api/like/{id}`，含 CSRF 头、防重复点击、成功提示）
- 平滑锚点滚动
- 评论区（加载 `/api/comments/{id}`、提交评论、字符计数、XSS `escapeHtml`、相对时间格式化）

### 9.2 `admin.js`（后台专属）
IIFE 封装，避免全局污染：
- `fetch` 封装 `request()`（自动附加 CSRF 头、统一错误处理、204 处理）
- 文章表格渲染 / 新建 / 编辑（集成 EasyMDE）/ 删除（二次确认弹窗）
- 修改密码弹窗（前端校验长度与一致性）
- 事件委托：`articlesTableBody` 上监听编辑/删除按钮

### 9.3 安全细节
- CSRF：从 `<meta name="_csrf">` / `_csrf_header` 读取，写入 `fetch` 的请求头
- XSS：所有用户内容（评论、文章标题等）经 `escapeHtml()` 转义后再 `innerHTML` 注入

## 12. 总结与改进建议

**优点**
- 片段复用（`fragments.html`）让导航栏/页脚/背景零重复维护，新增页面成本低
- 设计令牌统一管理，主题一致、易于换肤
- 原生技术栈，零构建、零前端依赖，部署简单

**可优化点**
1. ✅ **页脚已统一复用**：`article/detail.html` 与 `error/*` 已改为 `th:replace="~{fragments :: siteFooter}"`，邮箱/年份/站点名抽为可配置变量（`GlobalModelAdvice` + `application.yml` 的 `blog.site.*`）。
2. ✅ **详情页导航栏已复用** `siteNavbar` 片段，与首页/关于/登录/后台保持一致。
3. **硬编码内容**：关于页个人信息、技能百分比等仍为静态文本，可下沉到数据库/配置文件以实现后台可编辑。
4. **CDN 依赖**：RemixIcon / EasyMDE 走公网 CDN，生产环境建议自托管以提升稳定性与可离线性。
